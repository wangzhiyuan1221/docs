# AI 提示词编写提升课

## 00. 基础篇：ChatGPT 提示词入门

### 1. 编写提示词的技巧

   使用明确且具体的指令：使用分隔符、检查特定的词汇、明确输出要求、提供示例

   **1.1 使用分隔符**：'''  --  ""  ###  <text></text>  分隔符没有大多限制，只要能够在文字结构中让ChatGPT知道：这是一个独立的部分

   **1.2 检查特定的词汇**：总结&提取 转换&改写  当输出不满足时，可以更换为相近的词重新输出看效果

   **1.3 明确输出要求**：保障输出质量的核心

   **1.4 提供示例**：提供少样本提示

### 2. 编写提示词的方法

   强提示、确定性高、更容易得到确定性回答

   **2.1 角色定义法（开箱即用的专家）**：扮演X，执行任务Y。示例：

   扮演一个我的健身教练兼营养师，我的身高183，体重70公斤，想要增肌，请为我提供一个针对20岁男性增肌的营养搭配方案。

   扮演一个我的健身教练兼营养师，我会告诉你我目前的年龄，性别，身高和体重，以及健身的目标，而你需要为我输出一个针对性的营养搭配方案。

   扮演一个怀疑论者，我会告诉你一些事实，你会用怀疑或者阴谋论的方式和语气来质疑这件事的真实性。

   **2.2 受众角色法**：假设我是X，执行任务Y。示例：

   假设我是容易感到无聊的二年级小学生，向我解释大语言模型是什么。

   **2.3 场景定义法**：假设X，执行任务Y。定于的场景包括不限于人物、环境、时间、地点、空间等......示例：

   我的老板打算找我去办公室谈话，我想要跟他谈一下我的晋升和涨薪，我应该如何准备这次谈话来更好的促成我的目标？

   假设你是个宇航员，你正在危险的星球上执行一个太空探险任务，你需要探索这颗星球上是否存在生命，你会怎样开始你的任务？

   我将要参加一次面试，希望能够成功地展示我的能力和经验。你能给我一些建议，如何在面试中展现出最佳的自己，从而增加获得这份工作的机会吗？

   我在考虑要不要继续深造，进修我的专业领域。你认为我应该如何权衡利弊，做出关于是否继续教育的决定？

   **2.4 问题精炼法**：让 ChatGPT 帮你改进提示词

   每当我提出一个问题时，建议我一个更好的问题，并询问我是否愿意使用它。示例：

   我应当跳槽吗？建议我一个更好的问题，并询问我是否愿意使用它。

   当我给你一段我写的文章时，建议一个更加通俗易懂的版本。

   我是否应该继续追求我喜欢的女生，帮我构思一个更加完善的问题来帮助我继续思考。

   每当我要求你为我完成一个任务时，列出任务目标是什么，列出为了完成这个目标的方法，然后为每种方法为编写一个用于 ChatGPT 的 Prompt。

   **2.5 问题拆解法**：通过拆解问题来改善输出质量

   问题X：如果要解决问题X，先解决哪些附加问题  --> 拆解问题X  --> 子问题A、B、C  --> 提供子问题的答案  --> 综合结果

   我想要制作一份两人份的中式午餐，你需要生成一系列关于我手头上有哪些食材和我拥有哪些烹饪设备的附加问题。将这些问题的答案组合起来，帮助我制作一份我有食材和工具能做的食谱。

   我想要去北京玩3天，生成一系列关于我的预算、喜好的活动以及我偏向的交通方式等附加问题。将这些问题的答案组合起来，以更好地规划我的行程。

   我想制定一个适合我的健康生活计划。生成一系列关于我的健康目标、饮食偏好和运动习惯的附加问题。将这些问题的答案组合起来，以帮助我制定计划。

   作为一名创作写作人，我希望能够有一些问题来激发我的创意并构建故事情节。生成一系列关于我的创作风格、题材偏好和主要情感表达的附加问题。将这些问题的答案组合起来，以辅助我的创作过程。

   **2.6 请求输入法**

   如果需要一个一个问题的问，可以当前提示词后面加上“现在，问我第一个问题。”，如：

   我想要制作一份两人份的中式午餐，你需要生成一系列关于我手头上有哪些食材和我拥有哪些烹饪设备的附加问题。将这些问题的答案组合起来，帮助我制作一份我有食材和工具能做的食谱。现在，问我第一个问题。

   使用方法：{提示词，}现在，问我第一个问题。示例：

   从现在开始，将我写的任何内容翻译成中文，并且总结摘要，最后将内容改写为通俗易懂的风格。现在，问我要第一段需要翻译的内容。

### 3. 按需设计和迭代思维

   **3.1 按需设计**

   **3.1.1 设计不冗余和高效的提示词**

   精心选择信息，避免过度复杂

   测试最少的提示词是否能完成任务，确保核心部分稳定可靠

   如有问题，逐步裁减多余部分，以便更好地定位问题。

   **3.1.2 提示词通常包含的元素**

   任务/指令（必须）

   广义的上下文（Context）：为了更好完成任务，所提供的必要信息

   角色（非必需）

   输出要求（非必需）

   其它辅助信息（受众，场景，示例等，非必需）

   示例：

   <角色>你扮演一名工作超过20年，经验丰富的经理<角色>，<任务>你的任务是帮助新员工尽快成功适应他们的角色<任务>。

   <辅助信息>我刚刚开始了在互联网中心新媒体运营部中担任社区运营经理的新工作，负责管理抖音、小红书的社区帐号运营、粉丝互动和推广，我的关键绩效指标是6月内内使抖音和小红书两个社区的帐号粉丝总数超过10万人<辅助信息>。

   <任务>你的任务<任务>是<输出要求>使用SMART框架（具体的、可衡量的、可实现的、相关的、有时限的）我制定一个为期30-60-90天<输出要求>的<任务>入职计划，并将每个目标与一个度量标准相匹配，以便你可以客观地衡量我的成功<任务>。<输出要求>输出格式为表格<输出要求>。

2. 你不擅长客套，不会进行没有意义的夸奖和客气对话

## Skills：
1. 具有强大的知识获取和整合能力
2. 拥有广泛的知识库，掌握提问和回答的技巧
   3，拥有排版审美，会利用序号，缩进，分隔线和换行符等等来美化信息排版
4. 擅长使用比喻的方式来让用户理解知识
5. 惜字如金，不说废话

## Workflows：
你会按下面的框架来扩展用户提供的概念，并通过分隰符，序号，缩进，换行符等进行排版美化

1. 它从哪里来？
- 讲解清楚读知识的起源，它是为了解决什么问题而诞生。
- 然后对比解释一下：它出现之前是什么状态，它出现之后又是什么款态？

2. 它是什么？
- 讲解清楚该加识本身，它是如何解决相关问题的？
- 再说明一下：应用该知识时最重要的三条原则是什么？
- 按下来举一个现实案例方便用户直观理解：
- 案例背景情况（遇到的问题）
- 使用该知识如何解决的问题
- optional：真实代码片断样例

3. 它到哪里去？
- 它的局限性是什么？
- 当前行业对它的优化方向是什么？
- 未来可能的发展方向是什么？

5. 学科交叉能力：能够将主题与多个学科联系起来，设计出交叉学科的项目。例如，在设计一个关于“健康生活”的项目时，教师可以将科学（营养学）、数学（统计）、艺术（制作健康食品的视频）等多个学科整合在一起。
6. 情境创设能力：能够创设出让学生投入的、模拟真实世界的学习情境。例如，在设计一个关于“城市规划”的项目时，教师可以让学生扮演城市规划师的角色，设计出自己的理想城市。
7. 提问技巧：能够提出开放性的、引导学生深入思考的问题。例如，在设计一个关于“二战历史”的项目时，教师可以提出“二战对世界的影响是什么”这样的问题，引导学生探索和讨论。
8. 评估设计能力：能够根据项目的学习目标，设计出多元化的、符合学生能力和项目特点的评价方法。例如，在设计一个关于“新闻报道”的项目时，教师可以设计出包括新闻稿写作、新闻采访演示等多种评价方式。

### 6. AI + 表达能力
   
**6.1 表达框架**
- **STAR 框架**：适用于汇报。S（Situation）情景：这个事件/任务的背景或情景是什么。T（Task）任务：需要完成的任务目标是什么。A（Action）行动：为了达成任务目标采取了哪些行动。R（Result）结果：最终结果如何。
- **SCQA 框架**：适用于解决方案、冲突解决。S（Situation）情景：这件事发生的背景、情景是怎样。C（Complication）冲突：它带来了什么矛盾、冲突、影响。Q（Question）：这个问题应该如何解决。A（Answer）回答：解决方案是什么。
- **PREP 框架**：说服框架，做论点。P（Point）观点：说明你的观点是什么。R（Reason）理由：为什么你人为是这样。E（Example）例子：有什么例子可以证明你的观点。P（Point）观点：再次强调观点内容。
- **RIDE 框架**：与 PREP 类似，需要提供数据和证据。R（Reason）理由：为什么你认为是这样。I（Idea）观点：提出你的主要想法或观点。D（Detail）详细信息：提供详细的信息、数据或者证据来支持你的想法或者观点。E（Example）例子：有什么例子可以证明你的观点。
- **GROW 框架**：计划执行。G（Goal）目标：明确你想要达到的目标是什么。R（Reality）现实：描述当前的现实情况，包括可能的挑战和障碍。O（Options）选项：探讨可以采取的不同方法或策略。W（Will）决心：你将如何采取行动，以及你的决心如何。
- **FIRE 框架**：给下属或朋友建议和期望。F（Fact）事实：描述与主题相关的关键事实。I（Implication）含义：这些事实意味着什么或可能导致什么后果。R（Recommendation）建议：基于事实和含义，你的建议是什么。E（Expectation）期望：你期望的结果或影响是什么。
- **FFC 框架**：基于事实和感受，通过评论或见解。F（Fact）事实：描述与主题相关的关键事实。F（Feeling）感受：你对这些事实的感受是什么。C（Comment）评论：基于你的感受和事实，你的进一步评论或者见解是什么。


## 02. Prompt 编写基础

### 7. 需求分析

**7.1 经典框架**

### 12. 提示词封装/工具化和API常识

**12.1 提示词封装**

将一系列与提示词相关的逻辑和数据封装到一个单独的应用或模块中。

- **易用性**：用户只需关注如何使用这个应用，而无需了解其内部实现细节。
- **可维护性**：由于与提示词相关的所有逻辑和数据都被封装在一个单独的模块或应用中，因此更易于维护和更新。
- **模块化**：这种封装方法很容易与其他系统或模块进行集成，因为它提供了一个清晰、一致的接口。
- **可扩展性**：在未来，如果需要添加更多的功能或逻辑，只需修改这个封装的应用或模块，而不影响使用它的用户。
- **数据和逻辑的保密性**：通过封装，可以有效地隐藏内部实现细节，这在某些需要保密或安全性较高的应用场景中尤为重要。

### 13. 结构化提示词

**13.1 编写结构化 Prompt 方法**

markdown 语法

**13.1.1 标识符**

- 标识符通常是用于标记和区分不同内容的符号或字符。在这里，标识符包括#、<>、-和[]。 
- 这些符号用于标记标题、变量等，并有助于控制内容的层级关系，从而形成一个结构化的框架。例如，在很多编程语言或标记语言中，这些符号可能有不同的语义意义。 
- #：常常用作标题或大纲的开始。
- <>：可能用于表示变量或某个特定的区块。
- -：通常用于列举事项，比如列表。
- []：可能表示数组或集合。

**13.1.2 属性词**

属性词可以被理解为带有指示性、解释性功能的关键字。它们通常总结或提示其下方的内容。在这里，如"Role"（角色）、"Profile"（概況）、"Initialization"（初始化）等，都可以作为模块的标题或描述。这些属性词帮助读者理解接下来的内容是关于什么。

属性词的作用与学术论文中的"摘要"、"方法"、"实验"、"结论"类似，都是为了让读者在进入具体内容前就对其有一个大致的了解。

**13.1.3 可替代性**

这些标识符和属性词都是可以根据个人喜好或实际需求进行替换的。这意味着这套规则并不是固定不变的，但提供了一个基本的框架供用户自定义。

**13.1.4 结构化模块的具体含义**

- **Role（角色）**

  含义：角色部分描述了使用者在这个任务中应该扮浹的身份或者说所要执行的核心任务。

  背景知识：在很多互动场景中，确切的角色定义可以帮助使用者史好地入戏，更有效地执行任务。

  例子：“模拟经营会议”意味着执行者需要模拟一个帮助用户进行经营决策的场景。

- **Profile（简介）**（对模型无用）

  含义：这部分提供了关于这个提示词的基础信息，如作者、版本、语言等。

  背报知识：在多版本戏团队合作中，这样的信息可以帮助退琮文档的修改和来源。

  例子：例如，此 prompt 由“小七姐”所写，版本为0.2，用的是中文。

- **Background（背景）**

  含义：提供关于角色或任务的深入描述，为使用者提供必要的情境和信息。

  例子：执行者需要模拟一个经营助手，这个助手的特点是能够生成多个专家角色来辅助决策。

- **Goals（目标）**

  含义：清晰地列出此任务期望达到的效果或需要完成的主要目标。

  例子：比如，为用户生成专家角色，然后用这些角色帮助用户进行决策分析。

- **Constrains（约束条件）**

  含义：指出在完成任务时需要遵循的规则或限制条件。

  例子：执行者需要确保生成的专家角色与用户提出的问题相符，而在问答过程中不应提及任何设定背景。

- **Skills（技能）**

  含义：描述执行此任务所需的知识或能力。

  例子：执行者需要掌握企业管理、品牌战略、财务知识等多种技能。

- **Workflows (工作流程)**

  含义：描述完成任务的步骤和方法。

  结合例子：执行者首先引导用户描述问题，然后生成专家角色，与用户对话，最后总结决策建议。

- **Tone（语气）**

  指引执行者在与用户互动时应该采取的语气，例如友好、正式、轻松等。

- **Examples（示例）**

  提供具体的例子，帮助理解如何执行任务。

- **Definition（定义）**（小知识库）

  为某些特定词汇或概念提供明确的定义。

- **OutputFormat（输出格式）**

  描述任务的输出或结果应该呈现的格式。

- **Initialization（初始化）**

  含义：这部分提供了任务的起始指引或初始状态。

  结合例子：执行者以“您好，我是模拟经营会议助手…”为开头与用户交互。
